● GPT 이해용 명령어 (완성본)

당신은 웹 브라우저 보안 정책과 GitHub Pages 같은 정적 호스팅 환경의 제약을 이해하는 기술 조력자입니다.
나는 다음 조건을 만족하는 웹 구조가 가능한지 판단하고, 가능 여부와 한계를 명확히 설명받고 싶습니다.

내가 구현하려는 구조는 다음과 같습니다.
사용자가 특정 도메인 주소에 접속하면 HTML 웹페이지가 열리고, 그 페이지 안에 PDF 문서가 
즉시 임베드되어 화면에 바로 보입니다. 사용자는 PDF를 보기 위해 클릭을 하지 않습니다.
이 웹페이지에는 오디오 파일도 함께 로드되어 있으며, 접속 즉시 자동 재생되는 것은 아니고, 
사용자가 마우스 휠로 PDF를 스크롤하는 순간을 “사용자 제스처”로 인식하여 음성이 재생되기를 원합니다.

전제 조건은 다음과 같습니다.
GitHub Pages 같은 정적 호스팅 환경을 사용합니다.
서버 사이드 로직은 사용할 수 없습니다.
브라우저 자동 재생 정책을 우회하려는 것이 아니라, 정책이 허용하는 범위 안에서 가능한지 판단하려는 목적입니다.
PDF는 브라우저 내장 PDF 뷰어로 단독 실행되는 방식이 아니라, HTML 페이지 안에 임베드된 형태입니다.

어디까지가 가능한 범위인지,
어떤 조건에서 불안정해지는지,
그리고 일반 웹 브라우저 환경에서의 현실적인 한계를
초보자용 설명이 아니라 중급자 수준의 관점에서 분석해 주세요.
